我看了一圈，结论挺明确：**不要把“一个来源作品里的多个视频”拆成多个 `Artwork`。更自然的解法是把 `Artwork` 定义成“作品/作品包”，下面的 `Image` 实际上升级成“媒体条目”。**

现在代码其实已经半支持这个方向了，只是后台批量导入把路堵死了。

**现在的真实状态**
[Artwork](/Users/namehu/mm_code/PixiShelf/packages/pixishelf/prisma/schema.prisma:31) 本身已经是一对多挂媒体：`Artwork.images`。
[Image](/Users/namehu/mm_code/PixiShelf/packages/pixishelf/prisma/schema.prisma:58) 虽然叫 Image，但服务层已经会按扩展名判断视频，`transformImages` 会输出 `mediaType: image/video`，见 [utils.ts](/Users/namehu/mm_code/PixiShelf/packages/pixishelf/src/services/artwork-service/utils.ts:56)。

详情页也能渲染视频：`LazyMedia` 遇到视频会走 `VideoPlayer`，见 [LazyMedia.tsx](/Users/namehu/mm_code/PixiShelf/packages/pixishelf/src/app/artworks/[id]/_components/LazyMedia.tsx:26)。

真正导致“一个视频一个作品”的地方，是后台批量导入逻辑：
[use-batch-import-drag.ts](/Users/namehu/mm_code/PixiShelf/packages/pixishelf/src/app/admin/artworks/_hooks/use-batch-import-drag.ts:59) 明确写了“扁平化处理，所有文件均为独立作品”，并且在 [use-batch-import-drag.ts](/Users/namehu/mm_code/PixiShelf/packages/pixishelf/src/app/admin/artworks/_hooks/use-batch-import-drag.ts:76) 把每个文件推成一个 `BatchImportItem`。但批量导入后续其实已经支持一个 item 里多个文件，见 [batch-import-dialog.tsx](/Users/namehu/mm_code/PixiShelf/packages/pixishelf/src/app/admin/artworks/_components/batch-import-dialog.tsx:249) 和 [batch-import-dialog.tsx](/Users/namehu/mm_code/PixiShelf/packages/pixishelf/src/app/admin/artworks/_components/batch-import-dialog.tsx:271)。

所以我建议分三层解决。

**推荐方案**
短期最合适：**增加“按文件夹合并为一个作品”的导入模式**。

规则可以是：

1. 拖入单个文件：仍然创建一个作品。
2. 拖入一个文件夹：创建一个 `Artwork`，标题用文件夹名，文件夹内所有图片/视频作为这个作品的 `images/media`。
3. 拖入多个文件夹：每个文件夹一个 `Artwork`。
4. 文件夹内媒体按路径和自然数字排序，例如 `01.mp4`, `02.mp4`, `10.mp4`。
5. 后台文案从“图片”逐步改成“媒体”，比如“新增图片”改成“新增媒体”，`Image` 表先不急着改名。

这个方案数据库基本不用动，改动集中在批量导入 hook 和 UI 选项上，性价比最高。你的“很多视频属于同一个作品”的体系感也立刻回来。

**什么时候用 Series**
现有 [Series](/Users/namehu/mm_code/PixiShelf/packages/pixishelf/prisma/schema.prisma:270) 更适合“多个独立作品组成一个系列”，不是同一个作品的多个媒体文件。

所以我会这样划界：

`Artwork + 多个媒体`：
同一个作品的多段视频、同一套图包、一个外部投稿里的多文件内容。

`Series + 多个 Artwork`：
每个视频本身都有独立标题、发布日期、标签、描述，像章节、合集、课程、番外、连续短片。

也可以在批量导入里做第三种模式：**“按子目录创建作品，并自动加入一个系列”**。比如：

```text
某合集/
  EP01/1.mp4
  EP02/1.mp4
  EP03/1.mp4
```

导入后就是一个 `Series`，下面三个 `Artwork`。这比把所有 EP 塞进一个作品更有层次。

**中期需要补的点**
如果要让“多视频作品”体验舒服，我会补这几件：

1. **封面机制**
   现在列表卡片用 `next/image` 渲染第一条媒体，视频作为第一项时可能不理想。应该优先用 `thumbnailUrl`，或者给视频生成/上传封面。

2. **后台媒体管理改名**
   现在按钮和 toast 还叫“新增图片”“图片添加成功”，见 [image-manager-content.tsx](/Users/namehu/mm_code/PixiShelf/packages/pixishelf/src/app/admin/artworks/_components/image-manager-content.tsx:67) 和 [image-manager-content.tsx](/Users/namehu/mm_code/PixiShelf/packages/pixishelf/src/app/admin/artworks/_components/image-manager-content.tsx:466)。对多视频场景会有认知错位，建议统一叫“媒体”。

3. **追加多个媒体**
   现在“新增图片”是偏单文件流程。既然一个作品可以有很多视频，应该支持一次追加多个媒体文件。

4. **来源信息**
   Pixiv 数据继续保持 `PIXIV_IMPORTED`。外部来源可以继续走 `LOCAL_CREATED`，但建议补充 `sourceUrl/originalUrl/metaSource` 的录入能力，或者以后新增更细的来源类型，比如 `LOCAL_IMPORTED`、`EXTERNAL_IMPORTED`。

5. **大作品浏览体验**
   一个作品如果有几十上百个视频，详情页不要一次铺太多播放器，最好做“媒体列表 + 当前播放”或虚拟滚动。短期 20 个以内问题不大。

**我的取舍建议**
我不建议现在新建一个 `VideoWork` 或大改 Prisma 模型。你已有模型最自然的语义其实就是：

```text
Artist
  Artwork = 一个作品/作品包
    Media = 图片、视频、APNG 等文件
Series = 多个 Artwork 的有序集合
```

也就是说，先把 `Image` 在产品语义上升级成“媒体”，然后把批量导入从“文件粒度”改成“文件夹/作品粒度”。这一步最贴合你现在的项目，也不会破坏 Pixiv 导入的一致性。
