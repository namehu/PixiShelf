# 扫描策略清理 - 变更日志

## 版本 2.0.0 - 策略简化

### 🗑️ 移除的功能

#### 扫描策略
- ❌ **metadata** - 仅元数据扫描策略
- ❌ **media** - 仅媒体文件扫描策略  
- ❌ **full** - 完整扫描策略（两阶段）

### ✅ 保留的功能

#### 扫描策略
- ✅ **legacy** - 传统扫描策略，稳定可靠
- ✅ **unified** - 统一扫描策略，性能更好，**默认推荐**

### 🔄 API 变更

#### 类型定义
```typescript
// 变更前
type ScanStrategyType = 'metadata' | 'media' | 'full' | 'legacy' | 'unified'

// 变更后
type ScanStrategyType = 'legacy' | 'unified'
```

#### 错误处理
- 使用删除策略时返回 `400 Bad Request`
- 提供友好的错误信息和迁移建议
- 自动配置迁移到 `unified` 策略

#### API 响应
```json
{
  "statusCode": 400,
  "error": "Bad Request",
  "message": "Strategy 'full' is not supported. Available strategies: legacy, unified",
  "availableStrategies": ["legacy", "unified"],
  "suggestedStrategy": "unified"
}
```

### 🎨 前端变更

#### 策略选择界面
- 只显示 `legacy` 和 `unified` 两个选项
- 默认选择 `unified` 策略
- 更新策略描述文本

#### 用户体验
- 简化策略选择，减少用户困惑
- 提供清晰的策略说明
- 推荐标识显示最佳选择

### 🔧 技术改进

#### 新增组件
- **StrategyValidator** - 策略验证和错误处理
- **ConfigMigrator** - 配置自动迁移
- **MigrationLogger** - 迁移过程日志记录

#### 性能优化
- 减少策略选择开销
- 简化策略推荐逻辑
- 优化内存使用

### 📋 迁移指南

#### 对于开发者
1. **更新类型引用**: 确保代码中只使用 `'legacy'` 或 `'unified'`
2. **错误处理**: 添加对 400 错误的处理逻辑
3. **配置更新**: 检查配置文件中的策略设置

#### 对于用户
1. **自动迁移**: 系统会自动将旧策略迁移到 `unified`
2. **推荐使用**: 建议使用 `unified` 策略获得最佳性能
3. **向后兼容**: `legacy` 策略仍然可用，保证稳定性

### 🚨 破坏性变更

#### API 接口
- 使用 `metadata`、`media`、`full` 策略的 API 调用将返回 400 错误
- 前端界面不再显示这些策略选项

#### 配置文件
- 包含删除策略的配置会自动迁移
- 迁移过程会记录警告日志

### 📊 影响评估
